Setup a development environment

This page is dedicated to people who develop esup-lecture, intend to contribute, or want to use use-blank to start the development of a new web application.

Reminder: the development of esup-lecture is done under the Eclipse IDE, and obey quite strict rules (Checkstyle, Java compiler, Javadoc, RBE). Make sure that your Eclipse environment is properly configured before starting to code to keep the code as homogeneous as possible.

If you are a new developer of project esup-lecture or intend to contribute to the project, you need to:

  • create the project esup-commons
  • create the project esup-lecture

If you want to start the development of a new application, you have to:

  • create the project esup-commons
  • create the project esup-lecture
  • disconnect esup-lecture from its SVN repository
  • rename esup-lecture to something else
  • maybe connect it to your CVS or SVN repository.

All these operations are explained below.

Create the project esup-commons

Please refer to the explanations given on this page.

Create the project esup-lecture

Screenshots given on this page give you the main guidelines for the creation of the esup-lecture project.

At first, check out from the SVN repository, the Java nature of the project will be set later.

Now let us start with a Java project.

Note: at this step, there is nothing in the project. The build path will be configured later.

Checking out may take a little while.

Now all the files have been copied from the SVN repository, it is time to configure the project.

At first set the source directory.

In a first step, lt Eclipse compile in /bin, this will be changed later.

Now link to project esup-commons (previously created).

Now drag 'n drop file build-devel.xml into the Ant view,

And launch target _prepare to create output folder /build/WEB-INF/classes. Please note that configuration files build.properties and build-devel.properties must exist for the target _prepare to work; create them from the example files provided.

At least, refresh the project and set /build/WEB-INF/classes as the output folder of the Java compiler.

Your esup-lecture project is ready to be used. You can start the application (ant target _start) and use it.

If you are a esup-lecture developer, you are now ready to contribute.

If you intended to start the development of a new web application from esup-lecture, you have now to:

  • disconnect esup-lecture from its SVN repository
  • rename esup-lecture to something else
  • maybe connect it to your CVS or SVN repository.

Disconnect esup-lecture from its SVN repository

Rename esup-lecture to something else

The target rename-application is provided to rename esup-lecture automatically to a new name:

  • copy renameApplication-example.properties to renameApplication.properties
  • edit renameApplication.properties:
    org/esupportail/lecture=org/esupportail/myapp
    org.esupportail.lecture=org.esupportail.myapp
    edu/domain/lecture=edu/domain/myapp
    lecture-bugs=myapp-bugs
    ESUP-Portail\ Blank\ Application=ESUP-Portail My Application
    Welcome\ to\ the\ blank\ application\ of\ consortium\ ESUP-Portail=Welcome to my application
    Bienvenue\ dans\ l'application\ blanche\ du\ consortium\ ESUP-Portail=Bienvenue dans mon application
    project\ name\="esup-lecture=project name="esup-myapp
    logo_lecture=logo_myapp
    esup-lecture=esup-myapp
  • run ant rename-application
  • Refresh you project

Rename the project:

Eventually rename the base directory from c:/devel/esup-lecture to whatever you want (e.g. c:/devel/esup-myapp).

You may now connect your project to a new CVS or SVN repository and go on with your own business ;-)
