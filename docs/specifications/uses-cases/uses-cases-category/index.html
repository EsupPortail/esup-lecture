Étude de Cas : rafraichir_categorie

Action utilisateur : rafraichir une catégorie controllée.


Précondition :

catégorie définie dans le profil utilisateur & visible


Flux Normal :

Si la catégorie est controllée et d'accessibilité CAS alors appel de "recuperer_pt_cas", puis appel de "charger_categorie" en lui donnant le proxy ticket ; Evalue l'appartenance de l'utilisateur dans les groupes de visibilté de chaque source : Si il appartient à une source "forcée" abscente de son profil alors appel de "enregistrer_source" ; Si il appartient à nouvelle source "préabonnée" abscente de son profil alors appel de "enregistrer_source" ; Si une source invisible est dans le profil utilisateur alors appel de "effacer_source" . Si "cat.edit" devient <> ",managed,all" alors appel de "enlever_source" pour toutes les sources importées. Si "cat.edit" devient <> "personal,all" alors appel de "supprimer_source" pour toutes les sources crées/ajoutées.
