Étude de Cas : rafraichir_config_canal

Action globale.


Flux Normal :

Charge le fichier de config du canal en mémoire et appel de "charger_tous_profils_categories" puis de "charger_tous_contexts" au démarrage du canal et à l'ouverture d'une session utilisateur si le fichier de config a été modifié depuis son dernier chargement en mémoire. Vérifie que le fichier de mapping DTD-XSLT-xpath existe.


Flux Alternatif1 :

Erreur si le fichier de config du canal n'existe pas ou si le fichier de mapping non plus.

Étude de Cas : rafraichir_context

Action utilisateur : relit le context de l'utilisateur (= context global + personnalisations)


Précondition :

context chargé en mémoire


Flux Normal :

Evaluation de l'appartenance de l'utilisateur dans les groupes de visibilté de chaque catégorie : -- si il appartient à une catégorie "forcée", abscente de son profil alors appel de "enregistrer_categorie" ; -- si il appartient à une nouvelle catégorie "préabonnée" alors appel de "enregistrer_categorie" ; -- si une catégorie est dans le profil utilisateur alors qu'elle n'est dans aucun des groupes de visibilité alors appel de "effacer_categorie". Si "context.edit" devient <> "all,managed" alors appel de "enlever_categorie" pour toutes les catégories importées. Si "contexte.edit devient <> "all,personal" alors appel de "supprimer_categorie" pour toutes les catégories crées/ajoutées.
